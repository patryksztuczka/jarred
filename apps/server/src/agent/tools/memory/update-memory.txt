Updates the user's long-term memory by adding, removing, or replacing facts in memory files.

Available memory files:
- "profile" — basic info about the user (name, age, location, occupation, etc.)
- "projects" — information about projects the user is working on
- "facts" — miscellaneous facts, preferences, and notes about the user

Operations:
- "append" — add a new line/fact at the end of the file
- "remove" — remove a specific line that matches the given content exactly
- "replace" — replace a specific existing line with new content

Parameters:
- file: which memory file to update
- operation: "append", "remove", or "replace"
- content: the text to append (for "append"), the line to remove (for "remove"), or the new replacement text (for "replace")
- target: (required for "replace" only) the existing line to find and replace

When to use:
- When the user shares personal information (name, age, preferences)
- When the user mentions a project they're working on
- When the user corrects previously stored information
- When you learn something new about the user worth remembering

Guidelines:
- Store facts as concise bullet points (e.g., "- Name: John")
- One fact per line
- Do not store sensitive information like passwords or API keys
- Always confirm with the user before storing information, unless it's clearly intended to be remembered
